‎# 🤝 Contributing to SSHB Platform
‎
‎We welcome contributions from developers, designers, security experts, and anyone passionate about improving surveillance and intelligence systems for public safety.
‎
‎---
‎
‎## 🧭 Getting Started
‎
‎To contribute to this project:
‎
‎1. **Fork** the repository to your GitHub account.
‎2. **Clone** your forked repo locally:
‎   ```bash
‎   git clone https://github.com/YourUsername/SSHB-Platform.git
‎   ```
‎3. *Create a new branch* for your feature or fix:
‎   ```bash
‎   git checkout -b feature/your-feature-name
‎   ```
‎4. Make your changes and *commit* them with clear messages:
‎   ```bash
‎   git commit -m "Add: new report filtering logic"
‎   ```
‎5. *Push* your branch to GitHub:
‎   ```bash
‎   git push origin feature/your-feature-name
‎   ```
‎6. Open a *Pull Request* and describe your changes.
‎
‎---
‎
‎*🧪 Code Standards*
‎
‎- Use clear and descriptive variable names.
‎- Write comments for complex logic.
‎- Follow existing formatting and indentation.
‎- Avoid hardcoding sensitive data — use environment variables.
‎
‎---
‎
‎*🛡️ Security Guidelines*
‎
‎- Do not expose credentials or tokens.
‎- Validate all user inputs to prevent injection attacks.
‎- Use HTTPS for all API calls.
‎- Report any vulnerabilities via Issues or email.
‎
‎---
‎
‎*🧰 Areas You Can Contribute*
‎
‎- 🖥️ Frontend (React, Flutter)
‎- ⚙️ Backend (Node.js, Django)
‎- 🧠 AI Tagging (Python)
‎- 📄 Documentation
‎- 🧪 Testing & QA
‎- 🌐 Deployment & DevOps
‎
‎---
‎
‎*🐞 Reporting Bugs*
‎
‎If you find a bug:
‎
‎- Open an issue with a clear title and description.
‎- Include steps to reproduce the bug.
‎- Add screenshots or logs if available.
‎
‎---
‎
‎*💡 Suggesting Features*
‎
‎We love new ideas! To suggest a feature:
‎
‎- Open an issue labeled `enhancement`.
‎- Describe the feature and its potential impact.
‎- Share mockups or examples if possible.
‎
